# Phase 1 - Research / Discovery: issue-28

## Deliverable
Fix branch name duplication bug where `workflow/issue-N-issue-N` is generated instead of `workflow/issue-N` or `workflow/issue-N-<slug>` by:
1. Adding `project-name` subcommand to both copies of `kaola-workflow-roadmap.js`
2. Fixing `projectNameForIssue` in both copies of `kaola-workflow-claim.js` to read directly from the `.roadmap/issue-{N}.md` file
3. Refactoring `pickFirstActionableIssue` to call `projectNameForIssue` (DRY)
4. Adding `buildSinkBranchName` defensive helper used at both branch-construction sites
5. Adding regression tests in `simulate-workflow-walkthrough.js`

## Why
The silent `catch (_) {}` in `projectNameForIssue` masks a real wiring error (missing `project-name` subcommand). The fallback value `issue-{N}` is used as the project slug, which then gets concatenated with the `issue-{N}-` prefix already in the branch template, producing `workflow/issue-N-issue-N`. This is cosmetic now but masks a contract gap that could silently affect future claims.
